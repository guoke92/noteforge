| 入口 | 触发方式 |

## 3. 搜索类型对比

| 类型 | 后端 API | 匹配逻辑 | 适用场景 |
|------|----------|----------|----------|
| 全部 | searchFulltext | 标题 + 片段 + 标签 OR 匹配 | 默认 |
| 按文件名 | searchFulltext + 前端 filter | 过滤 title 包含 query | 已知文件名 |
| 按全文 | searchFulltext | 全文匹配 | 内容搜索 |
| 按标签 | searchFulltext + 前端 filter | 过滤 tags 包含 query | 标签浏览 |

## 4. 搜索结果的交互

### 4.1 关键词高亮

```
...NoteForge 的 <mark>Agent</mark> 记忆管理通过 IPC...
```

使用 `<mark>` 标签，背景色 `--color-warning/30`，文字色 `--color-text-primary`。

### 4.2 结果项

```
┌──────────────────────────────────────────────────┐
│  📄 agent-api.md                     ← 文件名    │
│  NoteForge 的 Agent 记忆管理...        ← snippet   │
│  路径: /MemLab/...   #agent  #ipc    ← 元数据    │
└──────────────────────────────────────────────────┘
```

| 区块 | 样式 | 说明 |
|------|------|------|
| 文件名 | font-medium | 直接打开目标 |
| Snippet | truncate, text-xs, text-secondary | 搜索上下文的片段 |
| 路径 | text-[11px], text-tertiary | 文件完整路径 |
| 标签 | tag-chip | 最多显示 3 个 |

### 4.3 点击行为

```
点击结果项 → setOpen(false) → openFile(r.filePath)
```

### 标签云交互

| 操作 | 行为 |
|------|------|
| 单击标签 | toggle 选中/取消 |
| 选中标签 | bg-accent text-white，字体按 count 比例缩放 (0.7rem ~ 1.1rem) |
| 多标签 AND 过滤 | 选中的标签做交集过滤 |
| 清除 | 清空所有选中标签 |
| 过滤输入 | 实时过滤标签列表 |
| 无匹配 | "无匹配标签" |

### 标签云视觉

```
#rag(12)          ← font-size: 1.1rem (最大)
  #agent(8)       ← font-size: 0.95rem
#architecture(4)  ← font-size: 0.85rem
  #design(3)      ← font-size: 0.8rem
    #rust(6)      ← font-size: 0.9rem
```

基于 `count / maxCount` 比例计算字号。

## 6. 状态矩阵

| 状态 | UI 表现 |
|------|---------|
| 空输入 | "输入关键词开始搜索 · Cmd+P 快速打开" |
| 搜索中 | 输入框右侧 "搜索中..." spinner text |
| 有结果 | 结果列表按 score 降序 |
| 无结果 | "未找到匹配的内容 · 尝试更换关键词" |
| 网络错误 | (N/A - 本地搜索，无网络依赖) |
