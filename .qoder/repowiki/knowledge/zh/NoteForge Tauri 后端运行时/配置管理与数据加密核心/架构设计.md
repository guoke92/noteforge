- **分层架构**：模块遵循 `models` (DTOs) -> `commands` (Tauri IPC Handlers) -> `core logic` (ConfigManager/EncryptionService) -> `infrastructure` (fs/rusqlite/crypto crates) 的分层设计。
- **状态管理**：通过 Tauri 的 `app.manage()` 将 `ConfigManager` 和 `Database` 注册为全局单例状态，在 `commands` 中通过 `tauri::State` 注入访问。
- **加密实现**：`encryption.rs` 实现了基于 PBKDF2 (SHA-256, 100k iterations) 密钥派生和 AES-256-GCM 的对称加密方案，用于工作区备份和 API 密钥存储。
- **错误统一**：`error.rs` 定义了 `NoteforgeError` 枚举，集成 `thiserror` 自动转换底层库错误（如 `rusqlite`, `io`, `serde_json`），并实现 `Serialize` 以便向前端返回结构化错误响应。
- **配置持久化**：`config.rs` 使用 JSON 文件存储应用配置，支持热更新与默认值回退。
