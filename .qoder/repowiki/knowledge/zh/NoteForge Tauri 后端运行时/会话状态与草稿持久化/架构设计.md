- 采用 Tauri State 管理模式：`WorkbenchSessionStore` 和 `WorkspaceDraftStore` 通过 `init_` 函数注册为全局单例，由 Commands 层通过 `State` 句柄调用。
- 分层架构：`commands/` 暴露 Tauri RPC 接口，`models/` 定义序列化载荷（如 `WorkspaceDraftPayload`），根目录下的 `.rs` 文件实现核心 I/O 逻辑。
- 草稿存储优化：`WorkspaceDraftStore` 实现分级存储策略，小文件存 JSON，超过 2MB 的大文件采用“原始内容 + 元数据侧车”格式，并辅以原子写（tmp + rename）保证数据一致性。
- 会话管理：`WorkbenchSessionStore` 负责保存和加载前端传来的不透明 JSON 会话数据，支持空值清理。
- 外部集成：`vault_watch.rs` 利用 `notify` 库在独立线程中监听文件系统变更，并通过 Tauri Event 系统向前端推送事件。
