- 采用 Tauri State 管理模式存储：`WorkbenchSessionStore` 和 `WorkspaceDraftStore` 在应用启动时通过 `init_` 函数初始化并注册到全局状态。
- 命令层（`commands/`）作为前端接口，直接调用存储层的 `save/load/delete` 方法，实现逻辑解耦。
- 保险库监听（`vault_watch.rs`）利用 `notify` 库在独立线程中运行，通过 Tauri 事件系统（`emit`）向 UI 推送文件系统变更，支持创建、修改、删除及重命名事件的映射。
- 数据持久化策略：会话数据保存为单一 JSON 文件，草稿数据根据 `vault_path` 的哈希值分片存储，均采用“写临时文件后原子重命名”的方式确保写入安全。
