- **分层架构**：遵循 `commands` (Tauri IPC入口) -> `models` (DTO/Schema) -> `repositories`/`scratch` (数据访问) 的分层设计。
- **数据存储策略**：采用混合存储模式，结构化数据（记忆、标签、时间线）通过 `rusqlite` 存储在 SQLite 中，由 `TagRepo` 和 `EmbeddingRepo` 封装访问；非结构化会话状态（草稿、面板布局）通过 `ScratchStore` 以 JSON 文件形式持久化到应用数据目录。
- **接口边界**：所有功能均暴露为 `#[tauri::command]`，通过 `State<Database>` 或 `State<ScratchStore>` 注入依赖，实现无状态命令处理与有状态存储管理的解耦。
